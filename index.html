Languages

This review introduces the concept of formal language in the context of Part II. Here, “language” broadly refers to a set of strings, with strings being sequences of symbols or tokens. Formal languages can represent anything from natural languages like English to computer programming languages.

Language
A language, denoted as LLL, is a set of strings formulated using a specified set of symbols Σ\SigmaΣ (characters or tokens).

Example: For English, the corresponding formal language comprises all possible sentences that can be formed with its words (tokens) and grammatical rules.

There are two important languages to know about.

The Empty Language ∅\emptyset∅
The language with no strings, denoted using the empty set. This is distinct from the empty string, a string of length zero represented by Λ\LambdaΛ.

The Universal Language Σ∗\Sigma^*Σ∗
The language of all possible strings from a symbol set Σ\SigmaΣ. It is defined as the universal set for a language over Σ\SigmaΣ. Meaning, all languages over a given alphabet Σ\SigmaΣ are a subset of Σ∗\Sigma^*Σ∗.

Immediate Issues

This broad definition, while very powerful, faces two immediate issues.

  1. Generation – How can we produce every possible string in a language starting from basic elements like an alphabet and rules?

Think about how difficult it would be to generate every possible English sentence. Where would you even start?

  1. Recognition – How do we determine if a given string is a valid member of a language, especially when a language can have an infinite amount of strings?

Throw iterating through an infinite set of strings to find a match out the window.

Solutions

The text introduces methods to address these challenges:

Regular Expressions (Addressed in Appendix 1)
A more concise, but non-general approach to directly represent (possibly infinite-length) languages.

Grammars
The primary method to generate (possibly infinite-length) languages.

Finite Automata
A finite way to recognize members of (possibly infinite-length) languages.

Grammars

Grammars use a set of rules to construct increasingly complex strings from simple elements. Grammars start with information you might expect to need for generating a language.

  1. An alphabet, Σ\SigmaΣ.
  2. A set of grammatical rules, PPP.

Grammars also incorporate variables or metasymbols (denoted as VVV). These symbols, distinct from the alphabet, are used only during string construction and must be replaced by the end. One variable is designated as the start symbol (usually SSS), marking the beginning of string generation.

Using variables, we can represent broad categories like nouns and verbs in production rules (e.g., NOUN → NOUN VERB), making sentence generation more efficient.

Grammar Formalism

Pulling it all together, a grammar, GGG, is defined by four elements: (VVV, Σ\SigmaΣ, SSS, PPP).

  • VVV: Set of variables/metasymbols.
  • Σ\SigmaΣ: The alphabet.
  • SSS: Start symbol.
  • PPP: Set of production rules.

Specific grammar types, like regular or context-free grammars, may limit the kinds of rules in PPP but still follow this basic structure.

An example of a grammar can be found in Appendix 2.

Finite Automata

Finite Automata address the problem of language recognition. Given a language LLL with alphabet Σ\SigmaΣ, an automaton specifies a design for a machine that takes in a stream of symbols from Σ\SigmaΣ, and outputs whether the current sequence is in LLL. It’s important to note that automata are purely design specifications, and not physical machines. However, examining and building plans still yields important insights.

Basic Components of Finite Automata: Finite Automata are composed of states and transitions:

States
These represent different conditions or situations that the automaton can be in. One or more of these states are designated as accepting states. If the automaton is in an accepting state (a state that signifies a successful recognition of the sequence), the input is considered part of the language LLL; otherwise, it is not. We also define a start state for the automaton, which is the state the automaton starts in when reading new input.
Transitions
Rules that determine how the automaton moves from one state to another based on the input symbol.

Operation:

  • The automaton starts in a designated start state.
  • It reads symbols from the input one at a time.
  • For each symbol, it follows the transition rule corresponding to its current state and the symbol, leading it to the next state.
  • If the automaton is in an accepting state, the input is considered part of the language LLL; otherwise, it is not.

Finite Automata Formalism

Pulling it all together, an automaton, MMM, is defined by five elements: (QQQ, Σ\SigmaΣ, q0q_0q0​, δ\deltaδ, AAA).

  • QQQ: A finite set of states.
  • Σ\SigmaΣ: The alphabet (the valid input symbols).
  • q0q_0q0​: The state from QQQ designated as the start state.
  • δ\deltaδ: The state transition function. It takes in the current state and the next input symbol, and outputs the resulting state.
  • AAA: The state(s) from QQQ designated as accepting states

Types of Finite Automata

  • Deterministic Finite Automata (DFA): Each state has exactly one transition for each symbol in Σ\SigmaΣ.
  • Nondeterministic Finite Automata (NFA): A state can have multiple transitions for the same symbol.

An example of a finite automaton can be found in Appendix 3.

Appendix 1: Regular Expressions

Regular expressions have foundational roots in the study of formal languages. They capitalize on the idea that languages are represented as sets, using set operations to condense languages.

Regular Expressions
Regular expressions are a method to describe sets of strings (languages) by condensing common patterns into a single expression. This expression is defined using a concise combination of symbols.

Regular expressions operate in much the same way math expressions do, simplifying a longer expression like 5×5×5×55 \times 5 \times 5 \times 55×5×5×5 to a shorter expression like 545^454.

Language Operations

Concatenation

The language L={ab}L = \{ab\}L={ab} can be represented by concatenating the two languages L1={a}L_1 = \{a\}L1​={a} and L2={b}L_2 = \{b\}L2​={b} as L1L2L_1 L_2L1​L2​.
{a}{b}\{a\}\{b\}{a}{b}
Regular expressions introduce a shorthand for this.
{a}{b}=ab\{a\}\{b\} = ab{a}{b}=ab
You can read this expression as “you need an aaa followed by a bbb for the string to be part of the language.” Therefore, the regular expression ababab completely describes the language LLL. Further shorthand exists for repeated concatenation (aaaaa=a5aaaaa = a^5aaaaa=a5).

Set Union

The language L={aba,ba}L = \{aba, ba\}L={aba,ba} can be represented by the union of two languages L1={aba}L_1 = \{aba\}L1​={aba} and L2={ba}L_2 = \{ba\}L2​={ba} as L1∪L2L_1\cup L_2L1​∪L2​.
{aba}∪{ba}\{aba\}\cup\{ba\}{aba}∪{ba}

Regular expressions introduce a shorthand for this.

{aba}∪{ba}=aba+ba\{aba\}\cup\{ba\} = aba + ba{aba}∪{ba}=aba+ba
You can read this expression as “you need either abaabaaba or bababa for the string to be part of the language.” Therefore, the regular expression aba+baaba + baaba+ba completely describes the language LLL.

Closure/Positive Closure

The language L={Λ,a,aa,aaa,aaaa,aaaaa,...}L = \{\Lambda, a, aa, aaa, aaaa, aaaaa, ...\}L={Λ,a,aa,aaa,aaaa,aaaaa,...} can be represented by using closure on the language L1={a}L_1 = \{a\}L1​={a} as L1∗L_1^*L1∗​.
{a}∗\{a\}^*{a}∗
Regular expressions introduce a shorthand for this.
{a}∗=a∗\{a\}^* = a^*{a}∗=a∗
You can read this expression as “you need zero or more instances of aaa for the string to be part of the language.” Therefore, the regular expression a∗a^*a∗ completely describes the language LLL.

Positive Closure

a+a^+a+
You can read this expression as “you need one or more instances of aaa for the string to be part of the language.”

Examples

Example 1

L={Λ,ab,abc,abcc,abccc,abcccc}L=\{\Lambda, ab, abc, abcc, abccc, abcccc\}L={Λ,ab,abc,abcc,abccc,abcccc}
RE=Λ+ab(c)∗RE = \Lambda + ab(c)^*RE=Λ+ab(c)∗

Either the empty string Λ\LambdaΛ or an ababab followed by zero or more ccc

Example 2

L={ab,ba,abab,baba,ababab,bababa,...}L=\{ab, ba, abab, baba, ababab, bababa, ...\}L={ab,ba,abab,baba,ababab,bababa,...}
RE=(ab)++(ba)+RE = (ab)^+ + (ba)^+RE=(ab)++(ba)+

One or more ababab or one or more bababa. Notice how we use positive closure because at least one instance of ababab or bababa needs to occur (There is no empty string Λ\LambdaΛ).

Appendix 2: Grammar Example

Using a grammar (VVV, Σ\SigmaΣ, SSS, PPP), I want to generate the language LLL of sentences that describe what my friend Aaron ate or drank at thanksgiving. To keep things concrete, let’s say the drinks were wine and cider and the foods were turkey and stuffing.

LLL = the set of English sentences that describe Aaron’s thanksgiving meal.

The Terminal Symbols Σ\SigmaΣ

Firstly, we’ll need Aaron’s name for the beginning of these sentences. We have the drinks and foods as wine, cider, turkey, and stuffing. Finally, we’ll include the verbs ate and drank So, my alphabet Σ\SigmaΣ, the set of possible symbols I want to appear in my final outcome is {Aaron, ate, drank, wine, cider, turkey, stuffing}.

Σ\SigmaΣ = {Aaron, ate, drank, wine, cider, turkey, stuffing}

The Variables VVV and The Start Symbol SSS

Now, let’s do the variables VVV. We’ll examine the statement

…the drinks were wine and cider and the foods were turkey and stuffing.

This statement hints at a very nice set of variables for us. We’ll call them FOOD and DRINK. And remember, we need a starting variable we’ll just call SSS

VVV = {FOOD, DRINK, S}
SSS = SSS

The Productions PPP

Okay, so now we have a good foundation of information. And remember, we need to start from the symbol SSS. SSS is the first “seed” in our generation, so we obviously need to have productions from SSS. I see two possible routes for the sentences to take.

S→S \rightarrowS→ Aaron ate FOOD.
S→S \rightarrowS→ Aaron drank DRINK.

Note the periods at the end.

We can’t stop here, since we have some non-terminal (production only) symbols in the form of FOOD and DRINK. We now need productions to eliminate these variables.

FOOD →\rightarrow→ turkey | stuffing (turkey or stuffing)
DRINK →\rightarrow→ wine | cider (wine or cider)

These four productions make up our set PPP, the final value in our quadruple. We now have a complete grammar.

Generating A Sentence

Given our grammar (VVV, Σ\SigmaΣ, SSS, PPP), we will generate one sentence as an example.

We need to start from SSS. Before our first step, we have the string

“SSS”

After using one of our productions (S→S \rightarrowS→ Aaron ate FOOD), we then have

“Aaron ate FOOD.”

After using another production (FOOD →\rightarrow→ turkey), we have

“Aaron ate turkey.”

Since there are no more productions we can perform, and the string is comprised only of terminal symbols from Σ\SigmaΣ, the sentence is completed.

Appendix 3: Automaton Example

Given Σ={a,b,c}\Sigma = \{a, b, c\}Σ={a,b,c}, consider the language LLL = {\{{The set of strings in Σ∗\Sigma^*Σ∗ containing at least one c}c\}c}. In order to construct an automata for this language, we need states and ways to transition between these states. We’ll start with the states. We need a start state and an accepting state at the very least (The start state is allowed to be the accepting state, by the way!). We’ll have the start state represent “I have not read a ccc yet” and have the accepting state represent "I have read at least one ccc". This gives us two states.

SSS, the start state (“I have not read a ccc yet”).
AAA, the accepting state ("I have read at least one ccc").

Now, we need transitions between these states. Obviously, if we read a ccc from any state, we want to go to the accepting state.

δ(S,c)=A\delta(S, c) = Aδ(S,c)=A
δ(A,c)=A\delta(A, c) = Aδ(A,c)=A

Once, we’re in the accepting state (we’ve read at least one ccc) no matter what the next character is, we’ve still read at least one ccc. All transitions from AAA lead back to AAA.

δ(A,a)=A\delta(A, a) = Aδ(A,a)=A
δ(A,b)=A\delta(A, b) = Aδ(A,b)=A

If we’re in the start state (we haven’t read a ccc) and we don’t read a ccc, we want to stay in the start state because we still haven’t read a ccc.

δ(S,a)=S\delta(S, a) = Sδ(S,a)=S
δ(S,b)=S\delta(S, b) = Sδ(S,b)=S

This automaton will, for any string in Σ∗\Sigma^*Σ∗, end up in either an accepting state or a non-accepting state, depending on whether it is in the language LLL = {\{{The set of strings in Σ∗\Sigma^*Σ∗ containing at least one c}c\}c}.
